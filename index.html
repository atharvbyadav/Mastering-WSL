Part 1: Introduction & Getting Started

What is WSL?

WSL (Windows Subsystem for Linux) is a compatibility layer developed by Microsoft that enables you to run a GNU/Linux environment directly on Windows—without needing a virtual machine or dual-boot setup.

With WSL, you can execute Linux commands, use open-source tools, and even run Linux applications side-by-side with your Windows workflow.

Why Use WSL?

  • Run Linux-native tools like bash, git, apt, and gcc natively on Windows.
  • Develop in a Linux environment while still accessing Windows files and software.
  • Lightweight alternative to full virtual machines or dual-booting.
  • Ideal for developers, sysadmins, and those learning Linux.

WSL 1 vs WSL 2

  • WSL 1: Uses a translation layer; faster startup and better for simple tasks.
  • WSL 2: Full Linux kernel inside a lightweight VM; better performance and compatibility (recommended).

Installing WSL (Recommended Method)

  1. Open PowerShell as Administrator
  2. Run: 
    wsl --install
  3. Restart your system if prompted.
  4. Once installed, you'll be asked to create a Linux username and password.

Note: This command sets WSL 2 as the default and installs Ubuntu as the default distro.

Checking WSL Version

wsl --list --verbose

This will display installed distros and whether they're using WSL 1 or WSL 2.

Set WSL 2 as Default (Optional)

wsl --set-default-version 2

Ensure future installations use WSL 2.

Installing Specific Distributions

wsl --install -d Debian

Replace Debian with the distro name of your choice (e.g., Ubuntu, Kali-Linux, etc).

Part 2: Managing Distros & Configuration

List Installed Distros

wsl --list --verbose

This lists all installed distros along with their versions and states.

Install a New Distro

wsl --install -d DistroName

Example: wsl --install -d kali-linux

Set Default Distro

wsl --set-default DistroName

New WSL sessions will start using this distro by default.

Convert Distro Version (WSL 1 <-> WSL 2)

wsl --set-version DistroName 2

Change 2 to 1 if you want to downgrade to WSL 1.

Unregister (Delete) a Distro

wsl --unregister DistroName

This will permanently delete the distro and its data. Be careful!

Warning: Unregistering a distro cannot be undone. All files and settings inside that distro will be lost.

Launch Specific Distro

wsl -d DistroName

Directly launches the chosen Linux distribution.

Set WSL Default Version (Global)

wsl --set-default-version 2

This sets WSL 2 as the default version for all future installs.

Open WSL Config File

To customize WSL behavior (like setting memory limits or auto-mounting drives), create or edit this file:

C:\Users\YourName\.wslconfig

Sample .wslconfig File:

[wsl2]
        memory=4GB
        processors=2
        localhostForwarding=true

This example limits WSL 2 to 4GB RAM and 2 CPU cores.

Part 3: Daily WSL Usage & Cool Tricks

Access Windows Files from WSL

cd /mnt/c/Users/YourName/Desktop

Navigate into your Windows drives easily via /mnt/c/, /mnt/d/, etc.

Open Files or URLs in Windows

explorer.exe .

This opens the current WSL directory in File Explorer.

Run Windows Apps from WSL

notepad.exe myscript.sh

Yes! You can launch Windows apps directly from your Linux terminal.

Use Linux Commands in PowerShell

wsl ls -la

Run WSL commands without launching the WSL shell.

Share Clipboard Between Windows & WSL

cat file.txt | clip.exe

This sends the output of file.txt to the Windows clipboard!

Use SSH Inside WSL

ssh user@hostname

Install your SSH keys inside WSL and securely connect to servers like a boss.

Running GUI Linux Apps (WSLg)

On WSLg (Windows 11), you can run GUI apps directly.

gedit

Opens the gedit GUI text editor right from the terminal!

Pro Tip: You can pin your favorite WSL apps or scripts to the Start Menu or taskbar!

Bonus: Add WSL to VS Code

Install the "Remote - WSL" extension in VS Code to open projects inside WSL seamlessly.

  • Open VS Code
  • Click Extensions & search Remote - WSL
  • Install and reload!

Back Up a WSL Distro

wsl --export DistroName backup.tar

Save your distro to a tarball. Later, you can restore it with --import.

Quick WSL Restart (Fixes bugs sometimes!)

wsl --shutdown

Closes all running WSL instances cleanly.

Heads up: WSL is rapidly evolving—keep your system updated to enjoy the latest features!
